# Execution Integrity Core 最小格式说明

## 目的
Execution Integrity 是用最小结构保证“执行过程可被独立验证、可发现篡改”的机制。

## 导出格式
`execution_log.json` 顶层固定为五个字段：

- `spec`：规范标识，用于声明该导出属于哪个结构族。
- `version`：格式版本号，当前示例为 `0.1.2`，用于区分兼容范围和升级边界。
- `exported_at`：导出时间戳，表示本次导出的生成时间。
- `hash_alg`：哈希算法标识，当前固定为 `sha256`。
- `chain`：执行事件链，包含每条记录及其哈希关联关系。

## 兼容原则
`0.1.x` 版本向后兼容，`0.2.0` 可能引入破坏性变更。
